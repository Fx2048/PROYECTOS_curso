## 🎯 Resumen Ejecutivo

**WILLAY** (quechua: *"Alerta"*) es una plataforma DeepTech de inteligencia climática diseñada para predecir heladas y sequías con **85% de precisión y 72 horas de anticipación**, protegiendo a más de **14 millones de pequeños agricultores andinos** vulnerables al cambio climático.

### 🔑 Propuesta de Valor
| Componente | Descripción |
|------------|-------------|
| **IA Predictiva** | Arquitectura BiLSTM para análisis de series temporales climáticas |
| **Fusión Satelital** | Integración de Sentinel-1 (SAR) + Sentinel-2 (Óptico) para superar nubosidad andina |
| **Hiperlocalización** | Alertas a nivel de parcela, no promedios regionales >10 km |
| **Offline-First** | Funcionamiento en zonas de baja conectividad con sincronización inteligente |
| **Accesibilidad** | Interfaz intuitiva + alertas SMS/Push adaptadas al usuario rural |

### 📊 Impacto Esperado
```
✅ Reducción del 40% en pérdidas por heladas/sequías
✅ Optimización del uso de agua mediante riego de defensa preventivo
✅ Fortalecimiento de la seguridad alimentaria en la región andina
✅ Democratización del acceso a tecnología climática de vanguardia
```

## 🚀 Stack Tecnológico

### 🖥️ Frontend & App
```
✅ React + TypeScript (Vite)
✅ Tailwind CSS + shadcn/ui
✅ Leaflet / Mapbox para visualización geoespacial
✅ MIT App Inventor / React Native para versión móvil
✅ PWA para funcionamiento offline
```

### ⚙️ Backend & IA
```
✅ Python + FastAPI para servicios REST
✅ TensorFlow / PyTorch para modelos BiLSTM
✅ Google Earth Engine API para procesamiento satelital
✅ Firebase / Supabase para base de datos y autenticación
✅ Docker para contenerización de servicios
```

### 🛰️ Datos & Sensores
```
✅ Sentinel-1 SAR + Sentinel-2 Óptico (ESA Copernicus)
✅ APIs meteorológicas: SENAMHI, NOAA, OpenWeather
✅ ESP32 + DHT22/BME280 para validación en campo
✅ Protocolos: Bluetooth LE, MQTT para IoT
```

### 🧪 Herramientas de Desarrollo
```
✅ Git + GitHub para control de versiones
✅ Figma para diseño colaborativo
✅ Notion para gestión de proyectos
✅ Google Colab para entrenamiento de modelos
```

## ⚙️ Instalación y Ejecución (MVP Local)

### 🔹 Requisitos Previos
```bash
# Node.js >= 18.x
node -v

# npm >= 9.x
npm -v

# Python >= 3.10 (para backend/IA)
python --version

# Git
git --version
```

### 🔹 Clonar y Configurar
```bash
# 1. Clonar el repositorio
git clone https://github.com/Fx2048/cultiva-seguro.git
cd cultiva-seguro

# 2. Instalar dependencias frontend
npm install

# 3. Configurar variables de entorno
cp .env.example .env.local
# Editar .env.local con tus claves de API (Mapbox, etc.)

# 4. Iniciar servidor de desarrollo
npm run dev
```

### 🔹 Ejecutar con Docker (Opcional)
```bash
# Construir imagen
docker build -t willay-mvp .

# Ejecutar contenedor
docker run -p 5173:5173 willay-mvp
```

> 📍 La aplicación estará disponible en: `http://localhost:5173`

## 🤝 Contribuciones

¡Las contribuciones son bienvenidas! Si deseas colaborar:

1.  **Fork** el repositorio
2.  Crea una rama para tu feature: `git checkout -b feature/nueva-funcionalidad`
3.  Realiza commits atómicos: `git commit -m 'feat: agrega validación de sensores'`
4.  Push a la rama: `git push origin feature/nueva-funcionalidad`
5.  Abre un **Pull Request** describiendo los cambios

```

fix:      Corrección de errores

style:    Cambios de formato (sin afectar lógica)
```
