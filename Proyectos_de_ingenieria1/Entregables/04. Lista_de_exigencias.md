      <p><b>Función Principal:</b> 🧠 Función Principal del Sistema El sistema WILLAY es una plataforma inteligente de alerta temprana orientada a la predicción de heladas y sequías en zonas agrícolas andinas, que integra sensores IoT en campo, datos satelitales multifuente y modelos avanzados de inteligencia artificial para generar alertas climáticas oportunas y confiables. El sistema está diseñado para alcanzar una precisión predictiva, basada en la fusión de datos locales (hiperlocales) y globales, permitiendo emitir alertas con una anticipación, lo cual es crítico para la toma de decisiones agrícolas preventivas. Además, WILLAY incorpora modelos de predicción de series temporales que permiten generar proyecciones climáticas, útiles para planificación agrícola estratégica (siembra, cosecha, gestión de recursos hídricos). La información es procesada automáticamente y entregada al usuario final mediante: Aplicativo móvil (interfaz principal) Notificaciones SMS y Push Soporte multilingüe: Español + Quechua De esta manera, WILLAY convierte datos complejos en alertas claras, accesibles y accionables, reduciendo pérdidas agrícolas y aumentando la resiliencia frente a eventos climáticos extremos.</b></p>

(4) Las pruebas deberán permitir validar el desempeño del sistema en condicion es reales y controladas.

Sobre los ensayos de durabilidad, el sistema deberá someterse a pruebas de resistencia (vibraciones, temperaturas entre -10°C y 45°C y humedad relativa entre 0% y 100%). La zona piloto deberá ser considerada únicamente para validación tecnológica inicial. 

El proyecto deberá contemplar pruebas futuras en regiones altoandinas para garantizar aplicabilidad real. Se deberán incluir zonas con condiciones de heladas como en la **Sierra de Lima (Huarochirí, Canta), Ayacucho y Huancavelica**.

        <li><b>Espacial:</b> Dominios de Extrapolación Tecnológica (TEDs) basados en similitudes biofísicas (clima, suelo, elevación). Si funciona en una zona andina, se replica a zonas similares sin re-validación completa.</li>
      </ul>

      <p><b>Validación y Métricas de Precisión:</b></p>
      <ul>
        
El sistema deberá validar su desempeño utilizando métricas estadísticas establecidas como el: coeficiente de determinación R²y el error cuadrático medio (RMSE). El modelo deberá alcanzar niveles adecuados de precisión conforme a estándares utilizados en estudios agroclimáticos [17].

El sistema deberá (1) incorporar el índice NDVI obtenido de imágenes satelitales para el monitoreo del estado de la vegetación [19], (2) intregar datos de sensores locales con datos remotos.

El sistema deberá ser validado con agricultores de zonas rurales altoandinas. La aplicación deberá diseñarse considerando limitaciones del entorno rural ( acceso limitado a internet, diferentes niveles de alfabetización digital). El sistema deberá cumplir con principios de accesibilidad, simplicidad e inclusión. El sistema deberá comunicar alertas de manera clara, oportuna y comprensible. Las alertas deberán cumplir con los siguientes criterios: uso de colores intuitivos (rojo, amarillo, verde), inclusión de mensajes simples y adaptación al lenguaje del usuario [20].

El sistema deberá demostrar impacto en la toma de decisiones (decisiones de siembra, implementación de medidas frente a heladas y reducción de pérdidas agrícolas). El sistema deberá cumplir con los siguientes umbrales mínimos de desempeño: Heladas: R² ≥ 0.82 y RMSE ≤ 1.2 °C (validación cruzada de 5 folds), Sequías: R² ≥ 0.78 y RMSE del SPI ≤ 0.3 y NDVI: correlación ≥ 0.85 con datos reales. Así mismo, el sistema deberá garantizar una tasa de entrega de SMS ≥ 95%. Por último, el sistema deberá mostrar intervalos de confianza en la interfaz de usuario para cada predicción.
